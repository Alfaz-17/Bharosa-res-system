Take design inspiration from PetPooja — India's leading restaurant POS. 
Key visual traits to replicate:

LAYOUT:
- Compact, information-dense layouts (not lots of empty space)
- Left sidebar navigation — always visible on desktop, collapsible icon 
  mode on smaller screens
- Top header bar: restaurant name, current user chip, notifications bell, 
  logout
- Content area uses full width — no narrow centered containers in staff UI
- Dashboard uses a grid of KPI stat cards + charts filling the whole screen

COLORS — Light Theme (default, also support dark):
- Sidebar bg: #1A1D2E (deep navy/charcoal like PetPooja)
- Sidebar text: #A0AEC0, active item: white text + #F97316 left border accent
- Main bg: #F5F6FA (light grey, not pure white)
- Card bg: #FFFFFF with box-shadow: 0 1px 4px rgba(0,0,0,0.08)
- Primary accent: #F97316 (orange — PetPooja's brand color)
- Secondary accent: #3B82F6 (blue for info states)
- Success: #10B981, Warning: #F59E0B, Danger: #EF4444
- Text primary: #1A202C, Text secondary: #718096
- Border: #E2E8F0

TYPOGRAPHY:
- Font: "Inter" (Google Fonts)
- Page titles: 20px/700
- Section headers: 14px/600 uppercase tracking-wide text-gray-500
- Body: 14px/400
- Table cells: 13px
- Stat numbers on KPI cards: 28px/700

COMPONENTS (PetPooja-style):
- KPI cards: left colored border-l-4, icon on right in colored circle, 
  big number + label + % change chip
- Status badges: pill shape, colored bg at 15% opacity + matching text
  CREATED=gray, CONFIRMED=blue, IN_KITCHEN=yellow, READY=green, 
  SERVED=teal, PAID=emerald, CANCELLED=red
- Tables: compact row height (40px), alternating very subtle row bg, 
  sticky header, action buttons appear on row hover only
- Sidebar items: 40px height, 14px, icon (20px) + label, 
  active = orange left border + white text + slightly lighter bg
- Modals: white, rounded-lg, 480px wide max, header with title + X, 
  footer with Cancel + Primary action
- Forms: label above input, input border #E2E8F0, focus ring orange, 
  helper text below in gray-500
- Buttons: primary = orange bg white text, secondary = white bg gray border,
  danger = red bg white text, all rounded-md h-9 px-4 text-sm font-medium
- Tabs: underline style, active = orange underline + dark text
- Search bars: icon inside, gray bg, no border until focused

KITCHEN DISPLAY:
- Dark background #0F172A
- Order ticket cards: dark gray bg #1E293B, white text
- Color urgency: <5min = normal, 5-10min = yellow border, >10min = red border
- Large readable font for item names

CUSTOMER MENU (QR Scan):
- Completely different vibe: warm, food-focused like Zomato/Swiggy
- White bg, food images prominent, card style menu items
- Sticky bottom cart bar with orange CTA button
- Clean category pills at top

BACKEND API — EXACT ENDPOINTS:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Base URL: process.env.NEXT_PUBLIC_API_URL

AUTH:
  POST /api/auth/login    → { email, password } → { accessToken, user }
  POST /api/auth/refresh  → (refreshToken cookie) → { accessToken }
  POST /api/auth/logout

MENU:
  GET/POST          /api/menu/categories
  GET/PUT/DELETE    /api/menu/categories/:id
  GET/POST          /api/menu/items
  GET/PUT/DELETE    /api/menu/items/:id

ORDERS:
  POST  /api/orders              → { table_number, notes, items: 
                                    [{menu_item_id, quantity, notes}] }
  GET   /api/orders              → ?status=&waiter_id=
  GET   /api/orders/:id
  PUT   /api/orders/:id/status   → { status }

BILLING:
  POST  /api/billing/:orderId/invoice
  GET   /api/billing/:orderId/invoice
  POST  /api/billing/:orderId/payments  → { method, amount, reference? }

ANALYTICS (ADMIN/MANAGER):
  GET /api/analytics/revenue
  GET /api/analytics/top-items
  GET /api/analytics/order-trends

USERS:
  GET    /api/users
  POST   /api/users
  PATCH  /api/users/:id
  DELETE /api/users/:id

MEDIA:
  POST /api/media/upload  (multipart)

RESTAURANT:
  GET /api/restaurant/settings
  PUT /api/restaurant/settings

WEBSOCKET:
  ws://[host]?token=ACCESS_TOKEN
  → broadcasts order events from Redis in real-time

AUTH HEADERS on every request:
  Authorization: Bearer <accessToken>
  X-Restaurant-Id: <user.restaurant_id>

    on second 401: clear store, redirect to /login
    on 403: show "Access denied" toast
    on 5xx: show "Server error, try again" toast
